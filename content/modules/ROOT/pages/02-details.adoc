== Timing and schedule

=== Full workshop (2 - 2.5 hours)

* **Module 1**: Tekton fundamentals (30 minutes)
* **Module 2**: Creating Tasks and TaskRuns (35 minutes)
* **Module 3**: Building your first pipeline (35 minutes)
* **Module 4**: Pipeline as code (30 minutes)
* **Module 5**: Monitoring and troubleshooting (20 minutes)

=== Abbreviated workshop (1 hour)

* **Module 1**: Tekton fundamentals (30 minutes)
* **Module 2**: Creating Tasks and TaskRuns (30 minutes, abbreviated)

== Technical requirements

=== Software versions

* Red Hat OpenShift {ocp_version}
* Red Hat OpenShift Pipelines {pipelines_version}
* OpenShift CLI (`oc`) version 4.20
* Tekton CLI (`tkn`) version 0.38 or later
* Web browser (Chrome, Firefox, Safari, or Edge)

=== Environment access

Participants need access to:

* OpenShift {ocp_version} multinode cluster (provided in the lab)
* Gitea Git server (provided in the lab)
* Web terminal accessible from the right panel in Showroom

=== Network requirements

* Internet connectivity is not required for this lab
* All required images and resources are pre-deployed in your lab environment

=== Workshop setup

In the **Terminal** tab, log in to OpenShift using your credentials:

[source,sh,role="copypaste"]
----
oc login https://api.%APPS_HOSTNAME_SUFFIX%:6443 -u %OPENSHIFT_USERNAME% -p %OPENSHIFT_PASSWORD%
----

NOTE: If the command does not work for CLI login, ask your facilitator for help.

Run the below command in the terminal to create the OpenShift project(namespace) you will use throughout this workshop:

[source,sh,role="copypaste"]
----
oc new-project tekton-workshop-%OPENSHIFT_USERNAME%
----

Expected output:

----
Now using project "tekton-workshop-%OPENSHIFT_USERNAME%" on server "...".
----

=== Setup validation

Run these commands in the terminal tab on the right to verify your environment is ready:

==== Verify OpenShift CLI version
[source,sh,role="copypaste"]
----
oc version --client
----

==== Verify Tekton CLI version

[source,sh,role="copypaste"]
----
tkn version
----
